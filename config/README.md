# ⚙️ config/

Runtime configuration for the `ups-battery-shutdown` service. One file per server, named by hostname.

```
config/
  ups-battery-shutdown.nut-pi.conf          ← Pi 1
  ups-battery-shutdown.homelab-nut.conf     ← Pi 2
```

Each file is written by `ups-service.sh` on the respective host and symlinked to `/etc/ups-battery-shutdown.conf` so the installed daemon can find it.

---

### `ups-battery-shutdown.<hostname>.conf`

Generated by `scripts/ups-service.sh` during setup. Symlinked to `/etc/ups-battery-shutdown.conf`.

```bash
# Re-run setup to regenerate:
sudo ./scripts/ups-service.sh setup

# Edit directly:
sudo ./scripts/ups-service.sh edit
```

**Options:**

| Key | Example | Description |
|---|---|---|
| `UPS` | `myups@localhost` | UPS name and host as reported by `upsc -l` |
| `THRESHOLD` | `50` | Battery % at which remote shutdown triggers (on battery only) |
| `REMOTE_NODES` | `"myuser@control-node"` | Space-separated list of `user@host` targets to SSH into |
| `POLL_INTERVAL` | `30` | How often (seconds) the daemon checks the UPS |
| `SSH_KEY` | `/root/.ssh/id_ed25519_ups` | Private key used for passwordless SSH to remote nodes |
| `REMOTE_SHUTDOWN_CMD` | `~/shutdown.sh` | Command run on each remote node when shutdown is triggered |
